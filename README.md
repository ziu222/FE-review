# CK — E-commerce frontend

Static storefront and admin UI for an e-commerce project: customer pages (browse, cart, auth) and an admin area (dashboard, products, orders, shops, users, finance, notifications). Built with plain HTML, CSS, and JavaScript; styles use [Tailwind CSS](https://tailwindcss.com/) v4.

## Requirements

- [Node.js](https://nodejs.org/) (for npm scripts and Tailwind CLI)

## Setup

```bash
npm install
npm run build:tw
```

`asset/css/tailwind-built.css` is generated by the Tailwind build and is listed in `.gitignore`. Run `build:tw` after clone or when you change `asset/css/tailwind-input.css` / Tailwind usage.

## Scripts

| Command        | Description                                      |
|----------------|--------------------------------------------------|
| `npm run build:tw` | Compile and minify Tailwind → `tailwind-built.css` |
| `npm run watch:tw` | Watch mode for Tailwind during development         |

## Local preview

This app is static HTML. Serve the repo root with any static server, for example:

```bash
npx serve .
```

Entry: `/` redirects to `customer/home.html` (see `index.html` and `vercel.json`).

## Deploy (Vercel)

The project includes `vercel.json`:

- **Build:** `npm run build:tw`
- **Output:** repository root (`.`)

Clean URLs are mapped to HTML files (e.g. `/admin/dashboard` → `admin/dashboard.html`). See `vercel.json` for the full rewrite list.

## Project layout

- `customer/` — Shop-facing pages (`home`, `product`, `cart`, `login`, `signup`, …)
- `admin/` — Admin pages (login, dashboard, management screens)
- `asset/css/` — Stylesheets and Tailwind input
- `asset/js/` — App logic (`core/`, `model/`, `user/`, `admin/`)
- `404.html` — Not found page

Auth and role helpers live in `asset/js/core/auth.js` (shared for admin, shop, and customer flows).
